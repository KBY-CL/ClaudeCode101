01

터미널이란?

한 줄 요약: 컴퓨터에게 글자로 명령을 내리는 도구입니다.

카페 키오스크 비유로 이해하기

카페에서 주문하는 두 가지 방법을 떠올려보세요.

  🖱️ GUI (키오스크 터치) → 화면에서 "아메리카노" 버튼을 누르고, "ICE" 버튼을 누르고, "결제" 버튼을 누른다
  ⌨️ CLI (점원에게 말로 주문) → "아이스 아메리카노 한 잔이요"라고 한마디면 끝

터미널 = 컴퓨터라는 카페의 점원입니다.
마우스로 이것저것 클릭하는 대신, 텍스트로 "이거 해줘"라고 말하면 컴퓨터가 바로 실행합니다. 처음에는 낯설지만, 익숙해지면 마우스보다 훨씬 빠르고 정확합니다.

⌨️

텍스트 기반 인터페이스

터미널은 글자만으로 컴퓨터와 소통하는 창입니다. 버튼이나 아이콘 대신, 명령어를 타이핑하면 컴퓨터가 결과를 텍스트로 보여줍니다. 마치 문자 메시지를 주고받는 것과 비슷합니다.

🛠️

개발자의 기본 도구

프로그래머들은 매일 터미널을 씁니다. 코드 실행, 파일 관리, 서버 접속, 패키지 설치 등 거의 모든 개발 작업이 터미널에서 시작됩니다. 개발의 "출입문"이라고 생각하면 됩니다.

🤖

왜 Claude Code에 필요한가

Claude Code는 터미널에서 실행되는 AI 도구입니다. "claude"라고 타이핑해서 시작하고, 대화도 터미널 안에서 진행됩니다. 터미널 기초만 알면 Claude Code를 바로 쓸 수 있습니다.

터미널은 어떻게 작동하나요?

터미널의 작동 원리는 매우 단순합니다:

1. 사용자가 명령어를 입력한다 (예: ls)
2. Enter를 누른다
3. 컴퓨터가 명령을 실행하고 결과를 보여준다
4. 다시 입력을 기다린다

이 과정의 반복입니다. 마치 카카오톡에서 메시지를 보내고 답장을 받는 것처럼, 명령을 보내고 결과를 받는 "대화"를 하는 겁니다.

02

터미널 여는 법

운영체제마다 터미널을 여는 방법이 조금 다릅니다. 아래에서 자신의 OS를 찾아보세요.

🍎 macOS
방법 1: Spotlight Cmd + Space 누르기 → "터미널" 입력 → Enter 방법 2: Finder 응용 프로그램 → 유틸리티 → 터미널.app
🪟 Windows
방법 1: 검색 시작 메뉴 클릭 → "PowerShell" 검색 → Enter 방법 2: 단축키 Win + X 누르기 → "터미널" 선택
🐧 Linux
단축키 한 방! Ctrl + Alt + T 바로 터미널이 열립니다. 대부분의 Linux 배포판에서 이 단축키가 작동합니다.
💜
처음 보는 검은 화면이 무서워도 괜찮습니다. 터미널은 그냥 텍스트 입력창일 뿐입니다. 잘못 입력해도 대부분 "그런 명령어 없어"라고 알려주기만 하고, 컴퓨터가 망가지는 일은 거의 없습니다. 편하게 이것저것 입력해보세요.
터미널을 열면 보이는 화면
# 이런 식으로 생긴 화면이 나타납니다 username@computer ~ $ █ # ↑ 여기에 커서가 깜빡이고 있습니다 # ↑ 이 "$" 또는 "%" 뒤에 명령어를 입력하면 됩니다 # ↑ "username"은 여러분의 컴퓨터 사용자 이름입니다
03

필수 명령어 5개

이 5개만 알면 터미널의 기본은 끝입니다. 하나씩 천천히 따라해보세요.

명령어 하는 일 비유 예시
pwd 현재 위치 확인 "지금 내가 어디에 있지?" 라고 물어보기 pwd → /Users/hyunho
ls 폴더 내용 보기 서랍을 열어서 안에 뭐가 있는지 보기 ls → Documents Photos Music ...
cd 폴더 이동 다른 방으로 걸어가기 cd Documents → Documents 폴더로 이동
mkdir 새 폴더 만들기 새 서랍 추가하기 mkdir my-project → my-project 폴더 생성
clear 화면 지우기 칠판을 깨끗하게 지우기 clear → 화면이 깨끗해짐
실전 예시: 명령어 따라해보기
# 1. 현재 위치 확인 $ pwd /Users/hyunho # 2. 현재 폴더에 뭐가 있는지 확인 $ ls Desktop Documents Downloads Music Pictures # 3. Documents 폴더로 이동 $ cd Documents # 4. 이동했는지 확인 $ pwd /Users/hyunho/Documents # 5. 새 폴더 만들기 $ mkdir my-first-project # 6. 만든 폴더로 이동 $ cd my-first-project # 7. 화면 정리하기 $ clear
💡
꿀팁: 명령어를 입력하다가 Tab 키를 누르면 자동완성이 됩니다. 예를 들어 cd Doc까지 입력하고 Tab을 누르면 cd Documents로 완성됩니다. 또한 위쪽 화살표(↑)를 누르면 이전에 입력한 명령어가 다시 나타납니다.
04

파일 시스템 이해하기

컴퓨터의 파일과 폴더가 어떻게 정리되어 있는지 알면, 터미널에서 길을 잃지 않습니다.

컴퓨터의 폴더 구조 (트리)
/ (최상위 폴더, 루트라고 부릅니다) ├── Users/ │ └── hyunho/ ← 여러분의 홈 폴더 (~) │ ├── Desktop/ ← 바탕화면 │ ├── Documents/ ← 문서 │ │ └── my-project/ ← 우리가 만든 폴더! │ │ ├── index.html │ │ └── style.css │ ├── Downloads/ ← 다운로드 │ └── Music/ ← 음악 ├── Applications/ ← 설치된 앱들 └── System/ ← 시스템 파일 (건드리지 마세요!)

서랍장 비유로 이해하기

파일 시스템은 거대한 서랍장과 같습니다.

최상위 폴더 (/) = 서랍장 자체. 모든 것이 여기 안에 있습니다.
하위 폴더 = 서랍장의 각 서랍칸. Documents, Downloads 같은 폴더들이죠.
파일 = 서랍칸 안에 들어있는 물건들. index.html, photo.jpg 같은 것들입니다.

터미널에서 cd 명령어는 "다른 서랍칸을 열겠다"는 뜻이고, ls는 "이 서랍칸 안에 뭐가 있지?"라고 확인하는 것입니다.

절대 경로 vs 상대 경로

📍 절대 경로 (전체 주소)
"서울시 강남구 역삼동 123번지" /Users/hyunho/Documents/my-project → 어디에 있든 정확한 위치를 가리킴 → 항상 /(슬래시)로 시작
🧭 상대 경로 (현재 위치 기준)
"여기서 오른쪽으로 두 블록" ./my-project (현재 폴더의 my-project) .. (한 단계 상위 폴더) ../Music (상위 폴더의 Music) → 현재 위치에 따라 도착지가 달라짐
🏠
홈 폴더 단축키: ~ (물결표)는 여러분의 홈 폴더를 의미합니다. 어디에 있든 cd ~를 입력하면 홈 폴더로 돌아갑니다. cd ~/Documents는 홈 폴더 안의 Documents로 바로 이동합니다.
05

개발 환경 설치

Claude Code를 사용하려면 몇 가지 도구를 먼저 설치해야 합니다. 터미널에서 하나씩 따라해보세요.

1

Homebrew 설치 (macOS)

Homebrew는 macOS의 "앱 스토어 터미널 버전"입니다. 개발 도구를 쉽게 설치하고 관리할 수 있게 해줍니다.

macOS 터미널
# Homebrew 설치 (이 한 줄을 복사해서 붙여넣고 Enter) /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # 설치가 끝나면 확인 brew --version Homebrew 4.x.x ← 이런 식으로 나오면 성공!
🪟
Windows 사용자: Homebrew 대신 winget이 기본 내장되어 있습니다. 또는 nodejs.org에서 직접 다운로드해도 됩니다.
2

Node.js 설치

Node.js는 JavaScript를 실행할 수 있게 해주는 프로그램입니다. Claude Code가 이것 위에서 작동합니다.

macOS 터미널
# macOS: Homebrew로 Node.js 설치 brew install node
Windows PowerShell
# Windows: winget으로 Node.js 설치 winget install OpenJS.NodeJS.LTS
3

설치 확인

모든 것이 잘 설치되었는지 확인합니다. 아래 명령어를 하나씩 입력해보세요.

설치 확인
# Node.js 버전 확인 $ node --version v20.x.x ← 숫자가 나오면 성공! # npm (Node 패키지 매니저) 확인 $ npm --version 10.x.x ← 이것도 숫자가 나오면 OK # 이제 Claude Code를 설치할 수 있습니다! $ npm install -g @anthropic-ai/claude-code
🎉
축하합니다! 여기까지 했다면 개발 환경 설정은 끝입니다. 이제 Claude Code를 설치하고 사용할 준비가 되었습니다. 다음 챕터에서 Claude Code가 무엇인지 자세히 알아보겠습니다.
06

Claude Code CLI 플래그

터미널에서 claude 명령어를 실행할 때 사용하는 주요 플래그들입니다. 공식 CLI 레퍼런스 기준 최신 내용입니다.

기본 실행 방법
# 대화형 세션 시작 (가장 기본) claude # 초기 프롬프트와 함께 대화형 세션 시작 claude "이 프로젝트를 설명해줘" # 출력 모드 (비대화형, 파이프 전용) claude -p "이 함수를 설명해줘" # 마지막 세션 이어서 시작 claude --continue # 세션 목록에서 선택해서 재개 claude --resume # 세션을 분기(포크)해서 재개 claude --continue --fork-session # ID 또는 이름으로 특정 세션 재개 claude -r "auth-refactor" "이 PR 마무리해줘"
모델 및 도구 제어
# 사용할 모델 지정 claude --model claude-sonnet-4-6 # 별칭으로도 지정 가능 (sonnet, opus) claude --model sonnet # 추가 작업 디렉토리 지정 claude --add-dir /path/to/other/project # 허용할 도구만 지정 (권한 프롬프트 없이 실행) claude --allowedTools "Bash,Read" # 특정 도구 차단 claude --disallowedTools "Write" # 사용 가능한 도구를 제한 (명명된 도구만) claude --tools "Bash,Edit,Read"
서브에이전트(Subagent) 정의
# --agents 플래그로 세션 전용 서브에이전트 정의 claude --agents '{ "code-reviewer": { "description": "코드 변경 후 자동으로 리뷰", "prompt": "당신은 시니어 코드 리뷰어입니다. 코드 품질, 보안, 베스트 프랙티스에 집중하세요.", "tools": ["Read", "Grep", "Glob", "Bash"], "model": "sonnet" }, "debugger": { "description": "오류 및 테스트 실패 디버깅 전문가", "prompt": "당신은 전문 디버거입니다. 오류를 분석하고 근본 원인을 파악해서 수정안을 제시하세요." } }'
시스템 프롬프트 커스터마이징
# 기본 시스템 프롬프트에 지침 추가 (권장) claude --append-system-prompt "항상 TypeScript를 사용해줘" # 전체 시스템 프롬프트를 교체 (고급 사용) claude --system-prompt "당신은 Python 전문가입니다" # 파일에서 시스템 프롬프트 로드 (인쇄 모드만) claude -p --system-prompt-file ./prompts/code-review.txt "이 PR 리뷰해줘"
출력 형식 및 자동화
# JSON 형식으로 출력 (스크립팅에 유용) claude -p "쿼리" --output-format json # 스트리밍 JSON 출력 claude -p "쿼리" --output-format stream-json # 최대 에이전트 턴 수 제한 (인쇄 모드) claude -p --max-turns 5 "쿼리" # 최대 예산 설정 (인쇄 모드) claude -p --max-budget-usd 5.00 "쿼리" # 기본 모델 과부하 시 폴백 모델 지정 (인쇄 모드) claude -p --fallback-model sonnet "쿼리"
플래그 설명 모드
--continue, -c 현재 디렉토리에서 가장 최근 대화 로드 전체
--resume, -r ID 또는 이름으로 특정 세션 재개, 또는 선택기 표시 전체
--fork-session --resume 또는 --continue 시 새 세션 ID 생성 (분기) 전체
--model 모델 지정 (sonnet, opus, claude-sonnet-4-6 등) 전체
--add-dir Claude가 접근할 추가 작업 디렉토리 추가 전체
--allowedTools 권한 프롬프트 없이 실행할 도구 지정 전체
--disallowedTools 모델의 컨텍스트에서 제거하고 사용 금지할 도구 전체
--tools Claude가 사용할 수 있는 내장 도구 제한 전체
--agents JSON으로 세션 전용 서브에이전트 동적 정의 전체
--print, -p 대화형 모드 없이 응답 출력 (파이프·자동화 용도) 인쇄 모드
--output-format 출력 형식 지정: text, json, stream-json 인쇄 모드
--max-turns 에이전트 턴 수 제한 (기본값: 무제한) 인쇄 모드
--max-budget-usd 중지하기 전에 소비할 최대 API 예산 (달러) 인쇄 모드
--system-prompt 전체 시스템 프롬프트를 사용자 정의 텍스트로 교체 전체
--append-system-prompt 기본 시스템 프롬프트 끝에 텍스트 추가 전체
--permission-mode 지정된 권한 모드로 시작 (예: plan) 전체
--mcp-config JSON 파일 또는 문자열에서 MCP 서버 로드 전체
--verbose 자세한 로깅 활성화 (디버깅에 유용) 전체
--debug 디버그 모드 활성화, 카테고리 필터링 지원 전체
--version, -v 버전 번호 출력 전체
--worktree, -w 격리된 git worktree에서 Claude 시작 전체
--remote claude.ai에서 새 웹 세션 생성 전체
--dangerously-skip-permissions 모든 권한 프롬프트 건너뜀 (주의해서 사용) 전체
💡
팁: --output-format json 플래그는 스크립팅 및 자동화에 특히 유용합니다. Claude의 응답을 프로그래밍 방식으로 파싱할 수 있습니다.
07

내장 슬래시 명령어

Claude Code 세션 안에서 /를 입력하면 사용 가능한 모든 명령어가 표시됩니다. 아래는 주요 내장 명령어 목록입니다.

ℹ️
사용 방법: Claude Code 세션 안에서 /를 입력하면 자동완성 목록이 나타납니다. / 뒤에 문자를 입력하면 필터링됩니다. 모든 명령어가 모든 사용자에게 표시되는 것은 아닙니다. 일부는 플랫폼, 요금제 또는 환경에 따라 다릅니다.
명령어 설명

08

파이프(Pipe) 활용

claude -p와 파이프(|)를 결합하면 터미널의 다른 명령어 출력을 Claude에게 바로 전달할 수 있습니다. 강력한 자동화 워크플로우를 만들 수 있는 기능입니다.

파이프(|)란?

파이프는 한 명령어의 출력을 다른 명령어의 입력으로 연결하는 터미널 기능입니다.

명령어A | 명령어B

명령어A의 결과가 자동으로 명령어B로 전달됩니다. claude -p는 비대화형 출력 모드이므로 파이프의 수신 측으로 완벽하게 작동합니다.

실전 파이프 예시
# 서버 로그를 실시간으로 Claude에게 보내서 오류 감지 tail -f server.log | claude -p "오류가 보이면 알려줘" # 변경된 파일 목록을 Claude에게 전달해서 보안 리뷰 git diff main --name-only | claude -p "변경된 파일들의 보안 이슈를 검토해줘" # 코드 파일을 읽어서 설명 요청 cat app.py | claude -p "이 코드를 설명해줘" # 새 문자열을 프랑스어로 번역해서 PR 생성 claude -p "새 문자열을 프랑스어로 번역하고 PR을 올려줘" # 로그 파일을 Claude에게 전달해서 분석 cat logs.txt | claude -p "이 로그를 분석해서 문제를 찾아줘" # 디렉토리 구조를 Claude에게 전달 find . -type f -name "*.ts" | claude -p "TypeScript 파일 목록을 보고 아키텍처를 설명해줘" # JSON 출력 형식으로 받아서 파싱 cat data.json | claude -p --output-format json "이 데이터를 요약해줘" | jq '.result'
⚡

자동화 스크립트에서 활용

CI/CD 파이프라인이나 쉘 스크립트 안에서 claude -p를 호출하면 AI 분석을 자동화된 워크플로우에 통합할 수 있습니다.

#!/bin/bash # 배포 전 보안 체크 자동화 git diff HEAD~1 | claude -p \ --output-format json \ "보안 취약점이 있으면 JSON으로 리포트해줘"
📊

JSON 출력으로 데이터 파싱

--output-format json을 사용하면 Claude의 응답을 다른 도구(jq 등)로 파싱해서 추가 처리할 수 있습니다.

claude -p "쿼리" \ --output-format json \ | jq '.result'
💡
핵심 차이: claude(인수 없음)는 대화형 세션을 시작하고, claude -p "..."는 응답만 출력하고 종료합니다. 파이프나 스크립트에서는 항상 -p 모드를 사용하세요.

터미널 최적화 팁

Claude Code를 더 쾌적하게 사용하기 위한 터미널 앱 추천과 설정 팁입니다.

🍎

iTerm2 (macOS 추천)

macOS 기본 터미널보다 훨씬 강력한 무료 터미널 앱입니다. Claude Code와의 호환성이 뛰어납니다.

Option을 Meta로 설정

설정 → 프로필 → 키 → Left/Right Option 키를 "Esc+"로 설정하면 Alt+B, Alt+F 등 단축키를 사용할 수 있습니다.

⚡

Warp (macOS/Linux)

AI 기능이 내장된 현대적인 터미널입니다. 명령어 자동완성과 블록 단위 출력이 특징입니다.

AI 자동완성 내장

Shift+Enter 지원을 위해 Claude Code 안에서 /terminal-setup을 실행해서 키 바인딩을 설치하세요.

🪟

Windows Terminal (Windows 추천)

Microsoft가 만든 현대적인 Windows 터미널입니다. 여러 탭과 PowerShell, WSL을 동시에 사용할 수 있습니다.

탭 지원 WSL 통합

Microsoft Store에서 무료로 다운로드할 수 있습니다.

Claude Code 인터랙티브 모드 키보드 단축키

단축키 설명
Ctrl+C 현재 입력 또는 생성 취소
Ctrl+D Claude Code 세션 종료 (EOF)
Ctrl+G 기본 텍스트 편집기에서 프롬프트 열기
Ctrl+L 터미널 화면 지우기 (대화 기록 유지)
Ctrl+O 상세 출력 토글
Ctrl+R 역방향 명령 기록 검색
Ctrl+B bash 명령 및 에이전트를 백그라운드로 실행
Ctrl+T 작업 목록 토글
Esc + Esc 되돌리기 또는 요약 (코드/대화를 이전 지점으로 복원)
Shift+Tab 권한 모드 토글 (자동 수락 ↔ Plan Mode ↔ 일반)
Option+P (Alt+P) 프롬프트를 지우지 않고 모델 전환
\ + Enter 여러 줄 입력 (모든 터미널에서 작동)
Shift+Enter 여러 줄 입력 (iTerm2, WezTerm, Ghostty, Kitty 기본 지원)
Option+Enter 여러 줄 입력 (macOS 기본값)
Up/Down 화살표 명령 기록 탐색
ℹ️
Option/Alt 키 단축키 설정 (macOS): Alt+B, Alt+F 등의 단축키를 사용하려면 터미널에서 Option을 Meta로 설정해야 합니다.
  • iTerm2: 설정 → 프로필 → 키 → Left/Right Option 키를 "Esc+"로 설정
  • Terminal.app: 설정 → 프로필 → 키보드 → "Option을 Meta 키로 사용" 체크
  • VS Code: 설정 → 프로필 → 키 → Left/Right Option 키를 "Esc+"로 설정

자주 묻는 질문

터미널을 처음 접하는 분들이 가장 많이 궁금해하는 것들

거의 같다고 보면 됩니다. 엄밀히 말하면 "터미널"은 텍스트 입출력 장치, "콘솔"은 컴퓨터에 직접 연결된 물리적 터미널, "셸(Shell)"은 명령어를 해석하는 프로그램을 의미합니다. 하지만 일상적으로는 셋 다 "글자로 명령을 내리는 검은 창"이라는 같은 의미로 사용됩니다. "터미널 열어", "콘솔 열어", "셸 열어" 모두 같은 뜻이라고 생각해도 무방합니다.
거의 그럴 일이 없습니다. 존재하지 않는 명령어를 입력하면 "command not found"(명령어를 찾을 수 없음)라고 알려주고, 아무 일도 일어나지 않습니다. 진짜 위험한 명령어(예: 시스템 파일 삭제)는 보통 관리자 권한(sudo)을 요구하기 때문에, 비밀번호를 입력하지 않는 한 실행되지 않습니다. 그러니 편하게 이것저것 입력해보세요. 실수해도 괜찮습니다!
둘 다 "셸(Shell)"의 종류입니다. 셸은 여러분이 입력한 명령어를 해석해서 컴퓨터에게 전달하는 통역사 같은 프로그램입니다. bash(Bourne Again Shell)는 오래되고 널리 쓰이는 셸이고, zsh(Z Shell)는 bash를 개선한 최신 버전입니다. macOS는 2019년부터 zsh가 기본이고, Linux는 보통 bash가 기본입니다. 초보자 입장에서 둘의 차이는 거의 느끼기 어렵고, 이 가이드의 모든 명령어는 둘 다에서 동일하게 작동합니다.
크게 세 가지 이유가 있습니다.

1. 속도: 마우스로 폴더를 열고, 파일을 찾고, 클릭하는 동안 터미널에서는 한 줄 명령어로 끝납니다. 예를 들어 100개의 파일 이름을 한번에 바꾸려면? GUI에서는 하나씩 클릭해야 하지만, 터미널에서는 한 줄이면 됩니다.

2. 자동화: 터미널 명령어들을 스크립트 파일에 적어두면, 복잡한 작업을 버튼 하나로 자동 실행할 수 있습니다. 반복 작업이 사라지죠.

3. 정확한 제어: GUI에서는 제공하는 옵션만 쓸 수 있지만, 터미널에서는 프로그램의 모든 기능에 접근할 수 있습니다. 마치 자동 vs 수동 기어 차이와 비슷합니다.

요약하면 이겁니다

필수 명령어 = pwd, ls, cd, mkdir, clear
핵심 가치 = 빠르고 정확한 컴퓨터 제어
