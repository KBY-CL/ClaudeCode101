01

MCP가 뭐야?

한 줄 요약: Claude에게 외부 세계와 소통하는 능력을 달아주는 표준 규격이다.

Claude에게 손발 달아주기

기본 Claude는 머리만 있는 상태입니다. 아무리 똑똑해도 혼자서는 GitHub에 PR을 올리거나, DB를 조회하거나, Slack에 메시지를 보낼 수 없죠.

MCP를 연결하면 Claude에게 손발과 눈이 생깁니다:

  🧠 머리 = Claude (추론, 판단, 계획)
  ✋ 손 = GitHub MCP (코드 관리, PR 생성)
  🦶 발 = Filesystem MCP (파일 탐색, 읽기/쓰기)
  👁 눈 = PostgreSQL MCP (데이터 조회, 스키마 파악)

이제 Claude는 "이 이슈 확인하고, 코드 고치고, PR 올려줘"를 한 번에 처리할 수 있습니다.

🔌

Model Context Protocol

MCP는 Model Context Protocol의 약자입니다. AI 모델이 외부 도구와 데이터에 접근하기 위한 개방형 표준 프로토콜이죠.

🔗

표준화된 연결 방식

USB처럼 하나의 규격으로 다양한 도구를 연결합니다. 각 도구마다 따로 연동 코드를 짤 필요 없이, MCP 서버만 있으면 바로 연결됩니다.

♾️

무한 확장 가능

GitHub, Slack, DB, 웹 검색, 파일 시스템 등 MCP 서버는 계속 늘어나고 있습니다. 직접 만들 수도 있어서 확장에 한계가 없습니다.

02

작동 원리

MCP가 어떻게 Claude와 외부 도구를 연결하는지 흐름을 살펴봅니다.

MCP 아키텍처 다이어그램
Claude Code MCP Client MCP Server External Tool | | | | | "GitHub 이슈 확인해" | | | | ————————→ | | | | | JSON-RPC 요청 | | | | ———————→ | | | | | API 호출 | | | | ——————→ | | | | | | | | 응답 데이터 | | | | ←—————— | | | 구조화된 결과 | | | | ←—————— | | | 결과를 자연어로 | | | | ←——————— | | | | | | |
1

사용자가 요청

"GitHub 이슈 #42 확인하고 수정해줘"처럼 자연어로 요청합니다. Claude가 어떤 MCP 서버를 호출할지 자동으로 판단합니다.

2

MCP Client가 요청 전달

Claude Code에 내장된 MCP Client가 JSON-RPC 2.0 형식으로 요청을 MCP Server에 전달합니다. 이 과정은 자동으로 이루어집니다.

3

MCP Server가 외부 도구 호출

MCP Server가 실제 외부 API(GitHub API, DB 커넥션 등)를 호출하여 데이터를 가져옵니다.

4

결과를 Claude에게 전달

외부 도구의 응답이 구조화된 형태로 Claude에게 돌아옵니다. Claude는 이 데이터를 분석하여 사용자에게 자연어로 답변합니다.

📡
기술 포인트: MCP는 JSON-RPC 2.0 프로토콜을 기반으로 합니다. 각 MCP 서버는 로컬에서 실행되는 독립 프로세스이며, Claude Code가 stdin/stdout 또는 HTTP를 통해 통신합니다.
03

인기 MCP 서버 카탈로그

가장 많이 사용되는 MCP 서버들입니다. 필요에 따라 골라서 연결하세요.

서버명 npm 패키지 주요 기능
GitHub @anthropic-ai/mcp-server-github 이슈/PR 조회, 생성, 코드 검색, 리포지토리 관리
Filesystem mcp-server-filesystem 파일 읽기/쓰기, 디렉토리 탐색, 파일 검색
Slack mcp-server-slack 메시지 읽기/보내기, 채널 관리, 스레드 검색
PostgreSQL mcp-server-postgres SQL 쿼리 실행, 스키마 조회, 테이블 분석
Brave Search mcp-server-brave-search 웹 검색, 최신 정보 조회, 뉴스 검색
Memory mcp-server-memory 대화 간 정보 저장, 장기 기억 관리, 지식 그래프
📦
더 많은 서버: MCP 서버 목록은 계속 늘어나고 있습니다. github.com/modelcontextprotocol/servers에서 공식 서버 목록을 확인할 수 있습니다.
04

설정 방법

MCP 서버를 Claude Code에 연결하는 두 가지 방법을 알아봅니다.

1

방법 A: CLI로 추가하기

가장 간단한 방법입니다. 터미널에서 한 줄 명령어로 MCP 서버를 추가합니다.

Terminal
# GitHub MCP 서버 추가 claude mcp add github -- npx -y @anthropic-ai/mcp-server-github # 환경변수와 함께 추가 claude mcp add github -- npx -y @anthropic-ai/mcp-server-github \ -e GITHUB_TOKEN=ghp_xxxxxxxxxxxx # Filesystem MCP 서버 추가 (특정 디렉토리만 접근 허용) claude mcp add filesystem -- npx -y mcp-server-filesystem \ /Users/me/projects
2

방법 B: JSON 직접 편집하기

설정 파일을 직접 수정하여 더 세밀한 제어가 가능합니다.

~/.claude/settings.json
{ "mcpServers": { "github": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-server-github"], "env": { "GITHUB_TOKEN": "ghp_xxxxxxxxxxxx" } }, "postgres": { "command": "npx", "args": ["-y", "mcp-server-postgres", "postgresql://localhost/mydb"] } } }
3

확인하기

서버가 제대로 연결되었는지 확인합니다.

Terminal
# 연결된 MCP 서버 목록 확인 claude mcp list # 출력 예시: github npx -y @anthropic-ai/mcp-server-github ✔ connected postgres npx -y mcp-server-postgres ... ✔ connected filesystem npx -y mcp-server-filesystem /Users/me/... ✔ connected
⌨️ CLI 추가
· 한 줄 명령어로 빠르게 추가 · 실수 가능성이 낮음 · 간단한 설정에 적합 · 서버 하나씩 추가할 때 편리
📝 JSON 편집
· 여러 서버를 한번에 설정 · 복잡한 환경변수 관리 가능 · 팀과 설정 파일 공유 용이 · 버전 관리(Git)에 적합
05

실전 활용 시나리오

MCP 서버를 연결하면 어떤 일이 가능해지는지, 3가지 실전 시나리오로 보여드립니다.

🚀 시나리오 1: GitHub 이슈 → 코드 수정 → PR 자동화

GitHub MCP 서버를 연결하면, 이슈 확인부터 PR 생성까지 한 번의 요청으로 자동화할 수 있습니다.

1

이슈 확인

Claude가 GitHub MCP를 통해 이슈 내용을 읽어옵니다.

2

코드 분석 & 수정

이슈와 관련된 코드를 찾아 수정합니다.

3

PR 생성

브랜치 생성, 커밋, PR 올리기까지 자동으로 처리합니다.

Claude Code 대화
You: GitHub 이슈 #42를 확인하고 수정해서 PR 올려줘 Claude: 이슈 #42를 확인했습니다. "로그인 페이지에서 이메일 유효성 검사 누락" 이슈네요. // Claude가 자동으로 수행하는 작업: 1. GitHub MCP → 이슈 #42 상세 내용 조회 2. 코드 분석 → src/auth/login.ts 파일 확인 3. 코드 수정 → 이메일 유효성 검사 로직 추가 4. GitHub MCP → 브랜치 생성, 커밋, PR 생성 Claude: PR #58을 생성했습니다. · 브랜치: fix/issue-42-email-validation · 변경 파일: src/auth/login.ts, src/utils/validators.ts · 테스트 추가: tests/auth/login.test.ts

🗄️ 시나리오 2: DB 스키마 분석 → 마이그레이션 작성

PostgreSQL MCP 서버를 연결하면, DB 구조를 파악하고 마이그레이션 파일을 자동 생성할 수 있습니다.

1

스키마 조회

PostgreSQL MCP를 통해 현재 DB 스키마를 읽어옵니다.

2

변경사항 분석

요청한 변경이 기존 데이터에 미치는 영향을 분석합니다.

3

마이그레이션 생성

안전한 마이그레이션 SQL과 롤백 스크립트를 작성합니다.

Claude Code 대화
You: users 테이블에 phone_number 컬럼 추가하는 마이그레이션 만들어줘 Claude: 현재 users 테이블 스키마를 확인했습니다. // PostgreSQL MCP를 통해 자동 수행: 1. 현재 users 테이블 스키마 조회 2. 기존 데이터 건수 확인 (100만 건) 3. 안전한 마이그레이션 전략 수립 Claude: 마이그레이션 파일을 생성했습니다. migrations/20240315_add_phone_number.sql · NULL 허용으로 추가 (기존 데이터 안전) · 인덱스 CONCURRENTLY 생성 (무중단) · 롤백 스크립트 포함

💬 시나리오 3: Slack 메시지 분석 → 요약 레포트

Slack MCP 서버를 연결하면, 채널의 메시지를 분석하여 요약 레포트를 자동 생성할 수 있습니다.

1

메시지 수집

Slack MCP를 통해 특정 채널의 최근 메시지를 가져옵니다.

2

내용 분석

논의된 주제, 결정 사항, 액션 아이템을 자동으로 분류합니다.

3

레포트 생성

정리된 요약 레포트를 마크다운으로 생성합니다.

Claude Code 대화
You: #dev-backend 채널의 이번 주 논의를 요약해줘 Claude: Slack MCP를 통해 이번 주 메시지 147건을 분석했습니다. ## 주간 요약 레포트 (#dev-backend) ### 주요 논의 사항 1. API v2 마이그레이션 일정 확정 (3/20) 2. Redis 캐시 TTL 정책 변경 논의 3. 신규 모니터링 도구 도입 검토 ### 결정된 사항 · API v2는 점진적 마이그레이션으로 진행 · Redis TTL은 1시간으로 통일 ### 액션 아이템 · @김개발: API v2 마이그레이션 가이드 작성 · @박서버: 모니터링 도구 PoC 진행
06

주의사항 & 트러블슈팅

MCP 서버를 사용할 때 반드시 알아야 할 사항들과 흔한 문제 해결법입니다.

🔑
환경변수 설정 필수: 대부분의 MCP 서버는 API 토큰이 필요합니다. GitHub MCP는 GITHUB_TOKEN, Slack MCP는 SLACK_BOT_TOKEN을 반드시 설정해야 합니다. 토큰 없이 연결하면 인증 에러가 발생합니다.
🛡️
보안 주의: MCP 서버는 로컬에서 실행됩니다. 이는 장점이자 주의점입니다. DB 서버에 접근하는 MCP를 설정하면 Claude가 실제로 쿼리를 실행할 수 있으므로, 프로덕션 DB에는 읽기 전용 계정을 사용하세요. 민감한 데이터가 있는 환경에서는 접근 범위를 최소화하는 것이 좋습니다.
⚠️
흔한 에러: "MCP server not responding"

이 에러가 발생하면 다음을 순서대로 확인하세요:
· npm/npx가 설치되어 있는지 — MCP 서버 대부분은 Node.js 기반입니다
· 패키지명이 정확한지 — 오타가 가장 흔한 원인입니다
· 환경변수가 올바른지 — 토큰이 만료되지 않았는지 확인
· 네트워크 연결 — 외부 API에 접근 가능한지 확인
💻
/mcp 커맨드: Claude Code 대화 중에 /mcp를 입력하면 현재 연결된 MCP 서버의 상태를 즉시 확인할 수 있습니다. 문제가 의심될 때 가장 먼저 실행해보세요.
트러블슈팅 명령어
# MCP 서버 상태 확인 claude mcp list # 특정 서버 제거 후 재추가 claude mcp remove github claude mcp add github -- npx -y @anthropic-ai/mcp-server-github # 설정 파일 직접 확인 cat ~/.claude/settings.json
07

자주 묻는 질문

MCP에 대해 가장 많이 궁금해하는 것들

네, 가능합니다! MCP는 오픈 프로토콜이기 때문에 누구나 서버를 만들 수 있습니다. TypeScript/Python SDK가 제공되어 있어서, 회사 내부 API나 특정 SaaS 도구를 MCP 서버로 감싸면 Claude가 직접 사용할 수 있게 됩니다. 예를 들어, 회사 내부 HR 시스템을 MCP 서버로 만들면 "이번 달 휴가 현황 알려줘"를 Claude가 처리할 수 있습니다.
MCP 서버는 필요할 때만 호출되므로, 많이 등록해두어도 전체 성능에 큰 영향은 없습니다. 다만 각 서버의 도구 목록(tool list)은 Claude에게 항상 전달되기 때문에, 극단적으로 많은 서버(20개 이상)를 등록하면 컨텍스트 윈도우를 차지할 수 있습니다. 실제 사용에서는 5~10개 정도가 적당한 수준입니다.
네! 프로젝트 루트에 .claude/settings.json 파일을 생성하고 MCP 설정을 넣어두면, 해당 프로젝트를 사용하는 모든 팀원이 동일한 MCP 서버 설정을 공유할 수 있습니다. 다만 API 토큰 같은 민감한 정보는 환경변수로 분리하고, 각 팀원이 자신의 토큰을 로컬에 설정하는 것이 좋습니다. 설정 파일은 Git에 커밋하되, 토큰은 .env에 넣어 .gitignore로 관리하세요.
물론입니다! MCP는 Claude Code의 기능을 확장하는 선택적 기능입니다. MCP 없이도 Claude Code는 파일 읽기/쓰기, 코드 생성, 터미널 명령어 실행 등 핵심 기능을 모두 사용할 수 있습니다. MCP는 "외부 서비스와의 연결"이 필요할 때 추가하는 것이라고 생각하면 됩니다. GitHub에 직접 PR을 올리거나, DB에 쿼리를 날리거나, Slack에 메시지를 보내는 등의 작업이 필요할 때 MCP를 도입하세요.

요약하면 이겁니다

MCP = Claude에게 외부 도구를 연결하는 표준 규격
설정법 = claude mcp add 한 줄이면 끝
핵심 가치 = 자동화 + 실제 시스템과 상호작용

시작이 어려우면, GitHub MCP부터 연결해보세요:
claude mcp add github -- npx -y @anthropic-ai/mcp-server-github
