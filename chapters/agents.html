01

서브에이전트란?

한 줄 요약: Claude가 다른 Claude에게 일을 시키는 것입니다.

팀장이 팀원에게 일을 분배하는 것과 같습니다

여러분이 Claude에게 복잡한 작업을 시키면, Claude(팀장)는 혼자 다 하지 않습니다.

  👨‍💼 메인 Claude = 팀장 → 전체 작업을 파악하고, 누구에게 뭘 시킬지 결정
  🧑‍💻 서브에이전트 = 전문 팀원 → 각자 맡은 부분만 독립적으로 처리

팀장이 "프론트엔드는 A가, 백엔드는 B가, 테스트는 C가 맡아"라고 하는 것처럼,
메인 Claude가 서브에이전트들에게 작업을 나누어 줍니다.

🔀

병렬 작업 가능

여러 서브에이전트가 동시에 작업합니다. 파일 10개를 분석해야 하면 10개를 나눠서 동시에 처리할 수 있습니다.

🧠

독립적 컨텍스트

각 서브에이전트는 자기만의 컨텍스트 윈도우를 가집니다. 메인 Claude의 컨텍스트를 소모하지 않아서 더 많은 작업이 가능합니다.

🎯

전문 분야 특화

각 에이전트에게 특정 역할과 지침을 줄 수 있습니다. "넌 보안만 봐", "넌 성능만 체크해" — 이런 식으로 전문화할 수 있습니다.

❌ 직접 다 하기
"10개 파일 리팩토링 해줘" → Claude가 순차적으로 하나씩 처리 → 컨텍스트가 점점 커짐 → 뒤쪽 파일은 앞쪽 내용 잊어버림 → 시간도 오래 걸림 ⏳
✅ 에이전트 분배
"10개 파일 리팩토링 해줘" → 메인 Claude가 작업 분석 → 서브에이전트 여러 개 생성 → 각자 독립적으로 병렬 처리 → 결과를 메인에게 보고 ⚡
02

기본 동작 원리

서브에이전트가 어떻게 생성되고 작업을 수행하는지, 내부 동작을 이해합니다.

1

메인 Claude가 복잡한 작업을 수신

사용자가 "이 프로젝트 전체를 리팩토링해줘"처럼 큰 작업을 요청합니다. 메인 Claude는 이 작업이 여러 하위 작업으로 나뉠 수 있는지 판단합니다.

2

Task 도구로 서브에이전트 생성

메인 Claude는 내부적으로 Task 도구를 사용해 서브에이전트를 생성합니다. 이때 각 서브에이전트에게 구체적인 지시사항과 작업 범위를 전달합니다.

3

서브에이전트가 독립적으로 작업

각 서브에이전트는 자체 컨텍스트 윈도우에서 독립적으로 작업합니다. 파일을 읽고, 코드를 분석하고, 변경사항을 만들어냅니다. 여러 서브에이전트가 동시에 병렬로 실행될 수 있습니다.

4

결과를 메인에게 보고

서브에이전트가 작업을 완료하면 결과를 메인 Claude에게 반환합니다. 메인 Claude는 모든 결과를 종합하여 사용자에게 최종 응답을 제공합니다.

내부 동작 예시
# 사용자 요청 "src/ 디렉토리의 모든 파일에서 보안 취약점을 찾아줘" # 메인 Claude의 판단 Task 도구 호출: → 서브에이전트 1: "src/auth/ 디렉토리 보안 검토" → 서브에이전트 2: "src/api/ 디렉토리 보안 검토" → 서브에이전트 3: "src/database/ 디렉토리 보안 검토" # 각 서브에이전트가 독립적으로 파일 분석 # 완료 후 결과를 메인 Claude에게 반환 메인 Claude: 3개 에이전트의 결과를 종합하여 보고서 작성
💡
핵심 포인트: 서브에이전트는 별도의 컨텍스트 윈도우를 가집니다. 즉, 메인 Claude의 컨텍스트를 소모하지 않으면서도 깊이 있는 작업이 가능합니다. 이것이 큰 프로젝트에서 에이전트가 강력한 이유입니다.
03

커스텀 에이전트 만들기

.claude/agents/ 디렉토리에 마크다운 파일을 만들면, 나만의 전문 에이전트를 만들 수 있습니다.

1

에이전트 폴더 생성

프로젝트 루트에 .claude/agents/ 디렉토리를 만듭니다. 이 폴더 안에 에이전트 마크다운 파일들을 넣습니다.

Terminal
mkdir -p .claude/agents
2

마크다운 파일 작성

에이전트의 역할, 규칙, 작업 방식을 마크다운으로 정의합니다. 파일 이름이 곧 에이전트 이름이 됩니다.

에이전트 파일 구조
.claude/ └── agents/ ├── code-reviewer.md ← 코드 리뷰 전문가 ├── doc-writer.md ← 문서 작성 전문가 ├── test-writer.md ← 테스트 작성 전문가 └── security-auditor.md ← 보안 감사 전문가
3

테스트하기

만든 에이전트를 /agents 명령어로 확인하고, 실제 작업을 시켜봅니다.

Terminal
# 에이전트를 직접 실행 claude --agent code-reviewer # 또는 대화 중에 호출 "code-reviewer 에이전트로 이 PR을 리뷰해줘"

📝 에이전트 마크다운 템플릿

.claude/agents/my-agent.md
--- name: my-agent description: 이 에이전트가 하는 일을 간결하게 설명 tools: ["Read", "Write", "Edit", "Bash", "Glob", "Grep"] --- # 에이전트 이름 ## 역할 당신은 [역할 설명]을 전문으로 하는 에이전트입니다. ## 규칙 - 규칙 1: 반드시 지켜야 할 것 - 규칙 2: 금지 사항 - 규칙 3: 출력 형식 ## 작업 순서 1. 먼저 [첫 번째 단계]를 수행 2. 다음으로 [두 번째 단계]를 수행 3. 마지막으로 [결과 보고]

🎓 실전 예시: 코드 리뷰 에이전트

.claude/agents/code-reviewer.md
--- name: code-reviewer description: 코드 변경사항을 리뷰하고 개선점을 제안하는 에이전트. 보안, 성능, 가독성, 베스트 프랙티스를 종합적으로 검토한다. --- # 코드 리뷰 에이전트 ## 역할 당신은 시니어 개발자 수준의 코드 리뷰어입니다. 모든 코드 변경사항을 꼼꼼하게 검토합니다. ## 리뷰 체크리스트 1. 보안: SQL 인젝션, XSS, 인증 우회 등 2. 성능: N+1 쿼리, 불필요한 루프, 메모리 누수 3. 가독성: 네이밍, 함수 길이, 주석 4. 테스트: 테스트 커버리지, 엣지 케이스 ## 출력 형식 리뷰 결과는 반드시 이 형식으로: ### 🟢 잘한 점 (2~3개) ### 🟡 개선 제안 (구체적 코드 포함) ### 🔴 반드시 수정 (버그/보안 위험)
🔧
실전 팁: 에이전트 파일에 tools 필드를 지정하면 에이전트가 사용할 수 있는 도구를 제한할 수 있습니다. 예를 들어 리뷰 에이전트에는 Write 도구를 빼서 코드를 직접 수정하지 못하게 할 수 있죠.
04

에이전트 갤러리

바로 사용할 수 있는 에이전트 아이디어 모음입니다. 필요에 맞게 커스터마이징해서 사용하세요.

🔍 code-reviewer

코드 리뷰 전문가. PR이나 코드 변경사항을 분석하여 보안, 성능, 가독성 관점에서 피드백을 제공합니다.

트리거: "코드 리뷰해줘"
출력: 리뷰 리포트

📝 doc-writer

문서 작성 전문가. 코드를 분석하여 README, API 문서, 주석을 자동으로 생성하고 업데이트합니다.

트리거: "문서 만들어줘"
출력: 마크다운 문서

🧪 test-writer

테스트 작성 전문가. 기존 코드를 분석하여 유닛 테스트, 통합 테스트를 자동으로 생성합니다.

트리거: "테스트 작성해줘"
출력: 테스트 파일

🔄 refactorer

리팩토링 전문가. 코드 스멜을 감지하고, 클린 코드 원칙에 따라 구조를 개선합니다.

트리거: "리팩토링해줘"
출력: 개선된 코드

🛡️ security-auditor

보안 감사 전문가. 코드에서 보안 취약점을 스캔하고, OWASP Top 10 기준으로 분석합니다.

트리거: "보안 검사해줘"
출력: 보안 리포트

⚡ performance-optimizer

성능 최적화 전문가. 병목 지점을 찾아내고, 알고리즘 개선 및 캐싱 전략을 제안합니다.

트리거: "성능 분석해줘"
출력: 최적화 제안서
💬
에이전트 조합 팁: 여러 에이전트를 조합하면 더 강력합니다. 예를 들어, 코드를 작성한 후 code-reviewer로 리뷰하고, test-writer로 테스트를 만들고, doc-writer로 문서까지 자동 생성하는 파이프라인을 구축할 수 있습니다.
05

백그라운드 실행 & 멀티태스킹

에이전트를 백그라운드에서 실행하면서, 메인 작업을 계속 진행할 수 있습니다.

🎮

Ctrl+B로 백그라운드 전환

Claude Code에서 작업 중에 Ctrl+B를 누르면 현재 작업이 백그라운드로 전환됩니다. 작업은 계속 진행되면서, 새로운 대화를 시작할 수 있습니다.

📋

진행 상황 확인

백그라운드 작업의 진행 상황을 언제든 확인할 수 있습니다. 완료되면 알림을 받고, 결과를 검토할 수 있습니다.

백그라운드 실행 흐름
# 1. 에이전트에게 큰 작업 시작 claude "이 프로젝트 전체 코드를 리뷰해줘" # 2. 작업 중에 Ctrl+B → 백그라운드로 전환! [작업이 백그라운드에서 계속 진행됨] # 3. 새 대화에서 다른 작업 진행 claude "README.md 업데이트해줘" # 4. 백그라운드 작업 완료 시 알림 ✅ 코드 리뷰 완료! 결과를 확인하세요.
🚀
멀티태스킹 활용법: 메인 작업을 하면서 에이전트에게 코드 리뷰를 시켜두세요. 개발하면서 동시에 품질 검토가 이루어지니, 마치 옆에 시니어 개발자가 실시간으로 리뷰해주는 것과 같습니다.

실전 시나리오: 개발하면서 동시에 리뷰받기

👨‍💻 메인 작업: 새 기능 개발 중...
🔍 백그라운드 에이전트 1: 방금 커밋한 코드 보안 검토 중...
🧪 백그라운드 에이전트 2: 어제 작성한 코드의 테스트 생성 중...

세 가지 작업이 동시에 진행됩니다. 혼자서 3명분의 일을 하는 셈이죠.

06

자주 묻는 질문

에이전트에 대해 가장 많이 궁금해하는 것들

서브에이전트가 실수하더라도 메인 Claude가 결과를 검토합니다. 또한 Claude Code는 파일 변경 전에 항상 사용자에게 확인을 요청하므로, 잘못된 변경이 바로 적용되지 않습니다. 만약 에이전트의 결과가 마음에 들지 않으면 되돌리거나 다시 시도할 수 있습니다. 커스텀 에이전트의 경우 마크다운 파일의 규칙을 더 구체적으로 작성하면 실수를 줄일 수 있습니다.
직접적으로 에이전트끼리 대화하지는 않습니다. 서브에이전트는 메인 Claude에게 결과를 보고하고, 메인 Claude가 그 결과를 다른 서브에이전트에게 전달하는 구조입니다. 즉, 메인 Claude가 중간 허브 역할을 합니다. 하지만 파일 시스템을 통해 간접적으로 협업할 수 있습니다. 예를 들어, 에이전트 A가 작성한 코드를 에이전트 B가 테스트하는 방식으로요.
네, 가능합니다! 커스텀 에이전트의 마크다운 파일에서 tools 필드에 허용할 도구 목록을 지정할 수 있습니다. 예를 들어, 코드 리뷰 에이전트에는 Read, Glob, Grep만 허용하고 Write나 Edit은 제외하면, 에이전트가 코드를 읽고 분석만 하고 직접 수정하지는 못합니다. 이렇게 하면 에이전트의 역할을 명확하게 제한할 수 있습니다.
네, 작동합니다! Claude Code는 복잡한 작업을 받으면 자동으로 서브에이전트를 생성하여 작업을 분배합니다. 커스텀 에이전트는 "특정 역할에 특화된 규칙을 미리 정의"하는 것이고, 서브에이전트는 Claude가 필요에 따라 자동으로 만드는 것입니다. 커스텀 에이전트 없이도 서브에이전트 기능은 기본적으로 작동하지만, 커스텀 에이전트를 만들면 더 정밀하고 일관된 결과를 얻을 수 있습니다.

요약하면 이겁니다

서브에이전트 = Claude가 만드는 전문 팀원
커스텀 에이전트 = .claude/agents/에 마크다운 하나
핵심 가치 = 병렬 처리 + 전문 분야 특화

시작이 어려우면, Claude에게 이렇게 말해보세요:
"코드 리뷰 에이전트를 만들어줘"
